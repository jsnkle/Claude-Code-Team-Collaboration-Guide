## Before Starting Work
1. Pull latest changes from main
2. Check for any outstanding PRs that need review
3. Review my assigned tickets

## Code Review Process
1. Read the PR description first
2. Check test coverage
3. Look for security issues
4. Review code quality
5. Leave constructive feedback

## Debugging Approach
1. Reproduce the issue first
2. Check the logs
3. Use breakpoints over console.log
4. Write a test to prevent regression

## End of Day
1. Commit work-in-progress to feature branch
2. Update ticket status
3. Note any blockers for tomorrow
